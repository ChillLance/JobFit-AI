# ADR-2026-002: Freeze speculative features; next slice = application kit, gated behind validation tracks

- Status: accepted 2026-07-16 (user approved the underlying review plan in-session)
- Scope: build priorities after the first-principles review of 2026-07-16
- Inputs: PRODUCT_DIRECTION_2026-07.md (desk research, admits no willingness-to-pay
  proof and names user validation as the first milestone), ADR-2026-001, and
  validation evidence #1 (`MEMORIES/user0-retrospective-2026-07.md`).

## Decision

1. **Purpose ranking confirmed by the owner**: primary = (a) product for other
   TW/HK working-holiday users and (b) portfolio/lead-gen; secondary =
   self-use (owner's Hokkaido contract is seasonal-leaning, so self-use
   returns within months) and AI-collaboration testbed.
2. **Work order**: Slice 0 (user #0 retrospective — done, evidence #1) →
   Slice 1 (portfolio track: public read-only demo deployment + README
   rewrite; no auth, demo data only) → Slice 2 (3–5 external
   working-holiday users run direction discovery + compare 2–3 real
   listings; success indicators per PRODUCT_DIRECTION §"Suggested success
   evidence").
3. **Feature freeze** until Slice 2 produces evidence: SearchIntent
   persistence UI, profile snapshot versioning, further extraction-pipeline
   hardening, new AI providers, auth/cloud/crawling (already gated by
   ADR-2026-001).
4. **Unfrozen by evidence #1**: the missing-info question list
   (JOB_EXTRACTION_DESIGN.md §7) — user #0 independently named "該問對方什麼
   問題" as decisive help.
5. **Next feature slice (after Slice 1), named "application kit"**: for one
   listing, generate a human-reviewed kit — (a) application message /
   resume-adjustment draft tailored to the listing's extracted conditions,
   (b) the questions-to-ask list from missing fields, (c) the savings figure
   with visible assumptions. This responds to user #0's strongest stated
   pain ("希望可以自動化依照各職缺條件後修改投遞").
   - This *amends* ADR-2026-001's stance that resume tailoring is only "a
     supporting feature": it remains **not the product position**
     (positioning stays "decision-and-application copilot"), but it is
     promoted from background support to the next validated build, because
     the only real user identified it as the decisive gap.
   - Guardrails unchanged and binding: every generated claim traces to a
     verified profile fact; nothing is ever auto-sent; human confirmation
     before any submission.
6. **Deferred, not rejected** (needs generalization evidence from Slice 2
   and/or a compliance design): social-post capture ("社群徵才po文海巡" —
   collection must remain attended), agency/employer reputation intel, offer
   -deadline comparison view.

## Consequences

- Positive: build effort now points at the two purposes the owner actually
  named, and the next feature has first real-user evidence behind it.
- Negative: single-user evidence can mislead; Slice 2 must check whether the
  application-tailoring pain generalizes before the kit grows beyond one
  listing → one draft.
- Supersedes/amends: amends ADR-2026-001 §Constraints (resume tailoring
  clause) as described in point 5. Everything else in ADR-2026-001 stands.
